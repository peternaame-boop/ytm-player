Section 4 of 7 — Hero + asset strategy

The 1.5MB GIF retires. Replaced with dual-source video, ~300-500KB total, with motion-reduced fallback.

What ships

  • Recut to 15-20s. The current 46s loop is too long — visitor attention is gone. Pick the four highest-impact moments: search → play → synced lyrics scroll → theme switch. Re-record asciinema-style or trim/splice from existing footage.
  • Dual-source <video>. AV1 (~300KB primary) + H.264 (~500KB fallback). Skip VP9 — AV1 supersedes it on Safari 17+/iOS 17.4+ which both ship hardware decode.
  • Poster frame as AVIF. Pre-rendered representative frame at ~2s in. This becomes the LCP element — paints before video loads. ~30KB.
  • prefers-reduced-motion fallback. If the visitor's OS reports motion-reduce, video doesn't autoplay — poster stays. CSS media query handles it, no JS needed.
  • Lives in public/media/ — Astro doesn't process; CF edge-caches with long TTL via _headers.

Concrete commands

# AV1 — primary
ffmpeg -i src.mp4 -an \
  -vf "fps=20,scale=1400:-2:flags=lanczos,format=yuv420p" \
  -c:v libsvtav1 -preset 4 -crf 38 -g 240 \
  -movflags +faststart \
  hero.av1.mp4

# H.264 — fallback
ffmpeg -i src.mp4 -an \
  -vf "fps=20,scale=1400:-2:flags=lanczos,format=yuv420p" \
  -c:v libx264 -preset slower -crf 28 \
  -tune stillimage -profile:v main \
  -pix_fmt yuv420p -movflags +faststart \
  hero.h264.mp4

# Poster (AVIF, ~2s frame)
ffmpeg -ss 2 -i src.mp4 -frames:v 1 \
  -vf "scale=1400:-2:flags=lanczos" \
  -c:v libaom-av1 -still-picture 1 -crf 30 \
  hero-poster.avif
// the actual <video> markup
<video autoplay muted loop playsinline
  poster="/media/hero-poster.avif"
  preload="metadata"
  width="1400" height="720">
  <source src="/media/hero.av1.mp4" type="video/mp4; codecs=av01.0.05M.08">
  <source src="/media/hero.h264.mp4" type="video/mp4; codecs=avc1.4d401f">
</video>
