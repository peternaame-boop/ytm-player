Notable composition decisions:
• Hero is short — title, tagline, video, install row, that's it. No CTAs button row competing with the install command itself.
• Mini comparison strip below the hero — 6 checkmarks in mono, single row. Replaces a heavier "features grid" section.
• Tabs always all-in-DOM via CSS only — Ctrl+F finds everything across all tabs without switching.
• Recent releases teaser on / shows latest 3, links to /changelog/. Full inline changelog NOT on the homepage anymore — too much weight, breaks page momentum (designer agent's point about 700 lines being a wall).
• Breadcrumbs on every non-/ route — needed for SEO BreadcrumbList schema and for visitor orientation.
• Prev/Next on docs and changelog routes — keeps people moving through related content without going back to an index.
