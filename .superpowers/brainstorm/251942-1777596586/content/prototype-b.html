▶ ytm-player

A full-featured YouTube Music player for the terminal. Vim keys, synced lyrics, mpv backend.

Install

Pick the channel that matches your setup. mpv is required system-wide on every platform.

PyPI (any platform)

pip install ytm-player

Arch Linux (AUR)

yay -S ytm-player-git

NixOS

Add the flake input from github:peternaame-boop/ytm-player and pull packages.ytm-player.

Gentoo (GURU overlay)

emerge media-sound/ytm-player

System dep on every platform: mpv. On Arch: sudo pacman -S mpv.

Configure

Configuration lives at ~/.config/ytm-player/config.toml on Linux/macOS or %APPDATA%\ytm-player\config.toml on Windows.

CLI

Subcommands talk to a running TUI via Unix socket IPC.

ytm                # launch the TUI
ytm play           # resume / play
ytm pause          # pause
ytm next           # skip
ytm now            # what's playing
ytm setup          # one-time auth
ytm doctor         # diagnostic dump for bug reports

Spotify import

Bring your playlists across.

ytm spotify import <playlist-url>

Uses the Spotify API when credentials are configured, falls back to the public scraper otherwise.

Troubleshoot

mpv not found

Install via your package manager. Arch: sudo pacman -S mpv. macOS: brew install mpv. Windows: see install docs for the libmpv-2.dll step.

Auth expired

Run ytm setup again to refresh cookies.

Get diagnostic dump

Run ytm doctor and paste the output into your bug report.

Architecture

Three layers:

  • services/ — backend singletons (Player, QueueManager, YTMusicService, etc.)
  • ui/ — Textual widgets (pages, sidebars, popups, widgets)
  • config/ — Settings dataclass, KeyMap with multi-key vim sequences

The app is split into mixins — one per concern (playback, navigation, keys, session, sidebar, track actions, MPRIS, IPC) all extending a typed host base.
