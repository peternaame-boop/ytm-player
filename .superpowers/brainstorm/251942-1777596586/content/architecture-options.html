Three architectures — pick how the URLs work

All three deliver C-level polish. They differ on whether the docs are one page or many, and what URLs exist.

URL map
/ → everything
/#install → tab
/#keys → tab
/#cli → tab
/#changelog → section
/#changelog-v1.9.2 → accordion
total indexable URLs: 1

1 — Pure single page

Your literal instinct. One URL. Tabs swap content via CSS only (all panels in DOM, Ctrl+F works, all visible to crawlers). Hash anchors auto-activate the right tab. Zero route management.

✓ Simplest to build & maintain
✓ Honors your "one page" framing exactly
✗ One ranking entity for everything (loses 4-5 SEO entities)
✗ No per-version changelog URLs to share
✗ No per-platform install URLs to rank
URL map
/ → landing + tabs
/install/arch → canonical
/install/macos → canonical
/install/nixos → canonical
/install/{4 more} → canonical
/changelog → index
/changelog/v1.9.2 → canonical
/changelog/{30+} → canonical
/vs/spotify-tui → comparison
/vs/cmus → comparison
total indexable URLs: ~45

2 — Hybrid recommended

"/" is your single-page tabbed UX exactly as you imagined — same hero, same tabs, same vibe. PLUS the same content also lives at canonical SEO routes, generated from the same markdown source. Visitors get tabs; Google gets entities; you author once.

✓ Single-page UX preserved on /
✓ ~45 indexed URLs targeting distinct intent
✓ Per-version changelog pages = 30+ long-tail captures
✓ Comparison pages convert at 5-10× branded queries
✗ More routing config (Astro handles it cleanly though)
✗ Need canonical hygiene (homepage canonical to itself, route canonical to itself)

3 — Two-page split

Classic docs-site structure (think bun.sh, helix-editor.com). "/" is pure landing — hero + features + install + "View docs →". Docs live behind a sidebar nav at /docs/*. Maximum SEO surface, but loses the single-page-with-tabs feel you described.

✓ Cleanest information architecture
✓ Matches what professional dev-tool sites do
✓ Maximum indexed surface
✗ Does NOT match your "one page, click yourself through tabs" framing
✗ More clicks for visitors to find anything
✗ Needs sidebar nav (more chrome)
My recommendation: option 2 (Hybrid). You're an SEO professional — losing 4-5 ranking entities to single-page architecture is real money on the table when this site is going to be your portfolio's most authoritative landing. Option 2 keeps your single-page tabbed UX exactly as you imagined while letting Astro auto-generate the canonical SEO routes from the same source markdown. Best of both. Authoring stays single-file.
