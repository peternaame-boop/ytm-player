deploy automation
Cloudflare Pages connected to ytm-player-site private repo.
Build cmd: pnpm install && pnpm build && pnpm pagefind
Output dir: dist/
Production branch: master · preview branches: any other branch builds to a *.pages.dev preview URL

Cross-repo trigger — a tiny GitHub Action lives on the public ytm-player repo: 
on:
  push:
    branches: [master]
    paths: ['docs/**', 'CHANGELOG.md']
  release:
    types: [published]

steps:
  - run: curl -X POST $CF_DEPLOY_HOOK_URL
On tag push: bump PINNED_TAG via a small commit to the website repo + trigger build.
On docs push between tags: trigger build with current tag (docs may be ahead of code).
stack summary
Astro 5 static export · Preact for islands · Pagefind for Cmd+K search · Shiki for code highlighting · @astrojs/sitemap for sitemap.xml · satori for OG image generation · remark-gfm + rehype-stringify for markdown · pnpm for package management · Cloudflare Pages for hosting · Cloudflare Web Analytics (cookieless) for traffic stats.

No CMS, no database, no serverless functions, no cookies of our own. ~52 static HTML files when built. Build time: ~30-45s.
