Section 6 of 7 — SEO hygiene + structured data + domain handoff

The boring-but-load-bearing layer. Set up once, ranks for years.

on-page hygiene (every page)
• <link rel="canonical" href="self"> on every route. Homepage canonical to /. Each /docs/* canonical to itself, NOT to the homepage tab. Google treats them as separate entities.
• Per-page <title> + meta description. Auto-derived from H1 + intro paragraph if not specified in content-config.
• OG + Twitter card meta tags on every page. Default OG image for /, per-version auto-generated for /changelog/v*.
• /sitemap.xml auto-generated by @astrojs/sitemap. /robots.txt hand-written (allow all, point at sitemap).
• Single H1 per page, sentence-case, contains the primary entity ("ytm-player").
structured data — what actually moves the needle
Ship:
  • SoftwareApplication on / and each /install/* — required keys: name, applicationCategory "MusicApplication", operatingSystem "Linux, macOS, Windows", offers.price "0", downloadUrl, softwareVersion (current), aggregateRating if you ever surface ratings
  • SoftwareApplication with softwareVersion per /changelog/v* page — distinct entity per release
  • BreadcrumbList on every non-/ page
  • sameAs array linking GitHub, PyPI, AUR, NixOS package, Gentoo overlay — binds the entity in Google's Knowledge Graph
  • SoftwareSourceCode on /architecture page (underused, helps AI Overviews cite correctly)
  • Person schema on author bio block, sameAs linking to naame.co — your EEAT spine for 2026
Skip:
  • FAQPage, HowTo — Google deprecated rich-result eligibility for non-medical/government FAQ in 2023, never reinstated. Pollution.
domain handoff — github → ytm-player.com
GitHub repo (DR ~93) currently dominates "ytm-player" branded SERPs. Won't outrank it for 6-12 months. Plays in priority order:
  1. Set "Website" field on the GitHub repo settings to https://ytm-player.com — this becomes the og:url on the GitHub repo page. Single biggest authority transfer signal you have.
  2. Update repo description to lead with "Official site: https://ytm-player.com" — first link is treated as homepage by Google.
  3. Update README.md top-of-file with a small "Visit ytm-player.com for full docs" badge linking the new site.
  4. naame.co → ytm-player.com link with anchor text "ytm-player" (never "click here") — best DR transfer you have.
  5. sameAs schema on / pointing to GitHub, PyPI, AUR — entity-binds in Knowledge Graph.
defensive registrations (your call, ~$30/yr)
Recommend grabbing: ytmplayer.com (no hyphen), ytm-player.dev, ytm-player.app. 301 redirect all to ytm-player.com canonical. Costs nothing to maintain on Cloudflare and kills typo-squatters / impersonators before they show up.
The sleeper play: the /vs/* comparison pages. Comparison-intent queries ("ytm-player vs cmus", "best terminal music player 2026") convert at 5-10× branded queries and Google rewards them in AI Overviews because they directly answer the "which should I use" question. Four pages, content already mostly available from the feature matrix. High ROI per page.
